# Sovereon Deployment Verification Checklist

## Pre-Deployment Requirements

### Environment Variables (Render Dashboard)

Required:
- [x] `DATABASE_URL` - Auto-set by Render PostgreSQL
- [x] `SESSION_SECRET` - Auto-generated by Render
- [x] `NODE_ENV` - Set to `production`
- [x] `PORT` - Set to `5000`
- [x] `FRONTEND_URL` - Set to `https://sovereon.vercel.app`

Optional (for email functionality):
- [ ] `RESEND_API_KEY` - Must be set manually in Render dashboard for contact forms to work
- [x] `FROM_EMAIL` - Defaults to `Sovereon <partners@sovereon.online>`
- [x] `ADMIN_EMAIL` - Defaults to `partners@sovereon.online`

### GitHub Secrets

Required for CI/CD:
- [x] `VERCEL_TOKEN` - Vercel authentication token
- [x] `VERCEL_ORG_ID` - Vercel organization ID
- [x] `VERCEL_PROJECT_ID` - Vercel project ID
- [x] `RENDER_SERVICE_ID` - Render service ID for deploy hooks
- [x] `RENDER_API_KEY` - Render API key for deploy hooks

## Post-Deployment Verification

### Backend Health Checks

1. **Basic Health**: `GET https://sovereon.onrender.com/api/health`
   - Expected: `{"status": "ok", "environment": "production"}`

2. **Database Health**: `GET https://sovereon.onrender.com/api/health/db`
   - Expected: All checks show `"status": "ok"`

3. **Email Health**: `GET https://sovereon.onrender.com/api/health/email`
   - Expected if Resend configured: All checks show `"status": "ok"`
   - Expected if Resend not configured: `"status": "error"` for config

### API Endpoints

4. **API Root**: `GET https://sovereon.onrender.com/api`
   - Expected: API info with list of available endpoints

5. **Public CMS Data**: `GET https://sovereon.onrender.com/api/public/team-members`
   - Expected: `{"success": true, "data": [...], "timestamp": "..."}`

6. **Admin Login**: `POST https://sovereon.onrender.com/api/admin/auth/login`
   - Test with credentials: admin / admin123
   - Expected on success: Session cookie set, user data returned
   - Expected on failure: `{"success": false, "error": {...}}`

### Frontend Verification

7. **Homepage**: `https://sovereon.vercel.app/`
   - Expected: Loads without errors
   - Content displays (team, services, testimonials)

8. **Admin Login Page**: `https://sovereon.vercel.app/admin/login`
   - Expected: Login form visible (not blank)
   - Can enter username and password

9. **CORS Check**:
   - Open browser console on frontend
   - No CORS errors when calling backend API

### Functionality Tests

10. **Contact Form** (if Resend configured):
    - Submit form on contact page
    - Data saved to database
    - Email notification sent to admin
    - Auto-reply sent to user

11. **Admin Login Flow**:
    - Login with admin/admin123
    - Session persists across page refreshes
    - Can access protected routes
    - Logout clears session

## Troubleshooting

### Backend Issues

**Problem**: Health check returns 503
- Check Render logs for startup errors
- Verify DATABASE_URL is set correctly
- Check if migrations ran successfully

**Problem**: CORS errors on frontend
- Verify FRONTEND_URL matches actual Vercel URL
- Check CORS configuration in backend

**Problem**: Session not persisting
- Check SESSION_SECRET is set
- Verify sameSite cookie setting for cross-domain
- Check if secure flag is correct for HTTPS

### Frontend Issues

**Problem**: Blank admin page
- Check browser console for JavaScript errors
- Verify VITE_API_URL is set correctly
- Check ErrorBoundary for error details

**Problem**: API calls failing
- Check Network tab for request failures
- Verify API URL in environment variables
- Check CORS configuration

## Rollback Procedures

### Backend Rollback
1. Go to Render Dashboard
2. Select the service
3. Go to "Deploys" tab
4. Find previous working deployment
5. Click "Manual Deploy" > "Deploy this commit"

### Frontend Rollback
1. Go to Vercel Dashboard
2. Select the project
3. Go to "Deployments" tab
4. Find previous working deployment
5. Click on it and select "Promote to Production"
